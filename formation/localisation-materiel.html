Localisation de matériel

Introduction

Projet de fin d'année de BTS réalisé en groupe de 3 et selon la méthodologie de travail du cycle en V. Le but du projet était de pouvoir retrouver un matériel spécifique sur un site. Pour ce faire, les utilisateurs possèdent une application de bureau depuis leur poste et une application web s'ils sont en déplacement avec un tablette. Ce matériel peut être localisé depuis un plan pour l'application bureautique. Chaque matériel possède une étiquette RFID qui sera lue à chaque fois que le matériel passe devant un porte possédant un lecteur RFID. Toute la gestion du matériel, salle, contrôleur et administrateur est faîte depuis une autre application de bureau à accès restreint. J'étais en charge de la base de données, de la gestion du matériel et des administrateurs ainsi que du site web.

Base de données

Le système de gestion de base de données choisi pour ce projet était MySQL. Mon rôle d'administrateur de base de données consistait à :

  • Ajouter un utilisateur avec des droits restreints à la base de données (aucun droit d'administration).
  • Créer les différentes tables nécessaires au fonctionnement du projet en se préoccupant des clés primaires, clés étrangères, champs uniques, etc.
  • Choisir le moteur de stockage et l'interclassement le plus approprié aux applications.
  • Réaliser les différentes requêtes pour accéder, ajouter, modifier ou supprimer les données souhaitées.
  • Coder la classe permettant à l'application l'accès à la base de données en installant au préalable de driver nécessaire.
  • Effectuer des tests de validation.
  • S'assurer que la mise en commun des différentes applications avec la base de données s'effectue facilement et sans aucun souci.

Application de gestion

L'application est faîte en C++ sous Qt Creator et selon l'architecture 3-tiers. Cette application permet de lire un badge et selon que le badge soit déjà enregistré ou non dans la base de données de pouvoir, ajouter, modifier ou supprimer le matériel dans la base. Chaque matériel est classé selon un type qui sera défini sur cette même interface. De plus, chaque matériel devait au cas par cas, être contrôlé de façon périodique. Cette période peut être précisée depuis cette application lors de l'ajout ou la modification d'un matériel afin de prévenir les utilisateurs quand le matériel doit être contrôlé. Pour finir, l'application permet d'ajouter d'autres administrateurs, mais aussi de modifier ou supprimer un administrateur (l'ancien mot de passe de cet administrateur sera alors demandé par sécurité).

Site web

L'application web a été réalisé avec Java EE et plus précisément avec la librairie JSTL en adoptant le patron de conception MVC. Le site permet de faire différentes recherches sur le matériel selon un critère spécifique. Les matériels devant être contrôlé avant utilisation est affiché sous une autre couleur afin de connaître ces différents matériels en un seul coup d'œil. Une page peut au maximum afficher une liste de 10 matériels. Si la demande excède ce nombre, une barre de pagination apparaitra et le remplissage de cette liste se fera en Ajax depuis des informations envoyées en JSON par une Servlet.
